# Reglas para agentes de IA

Este documento define reglas obligatorias para cualquier agente de IA que participe en cambios del proyecto.

## Regla principal

Antes de modificar codigo, el agente debe leer los documentos .md dentro de la carpeta rules.

## Requisitos minimos de trabajo

1. Leer este archivo completo antes de proponer o aplicar cambios.
2. Revisar otros .md en rules si existen.
3. Revisar rules/Colaboracion.md para respetar responsables y zonas compartidas.
4. Respetar la estructura de carpetas definida del proyecto.
5. Evitar cambios fuera del alcance solicitado.
6. Documentar cambios relevantes en README.md cuando afecten organizacion o flujo.

## Validacion previa a editar

- Confirmar que el requerimiento del usuario esta claro.
- Confirmar que las reglas de rules no se contradicen con la solicitud.
- Confirmar si la carpeta a modificar tiene un responsable asignado en rules/Colaboracion.md.
- Si hay conflicto entre solicitud y reglas, pedir aclaracion antes de aplicar cambios.

## Politica de cumplimiento

Si un agente no ha leido rules/Agent.md, no debe modificar codigo.
